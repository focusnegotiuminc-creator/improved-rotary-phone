Routing Lanes

Default path for the combined engine UI.

  1. Lane 1 · Intake and task normalization OpenAI prepares the first structured brief and labels the work by surface and urgency.
  2. Lane 2 · Long-context review Anthropic reviews the brief for coverage, missing context, and risk framing.
  3. Lane 3 · Local execution Local workers handle deterministic transforms, parsing, and file outputs.
  4. Lane 4 · Memory update and publish path The continuity layer captures decisions, artifacts, and next actions for the next run.

Implementation notes

Initial UI scope for the combined engine build.

  1. Provider registry A configurable registry for OpenAI, Anthropic, local runners, and memory adapters.
  2. Router policies Explicit profiles that decide who handles synthesis, code, verification, and publishing.
  3. Session state A state model that preserves mission, provider assignments, artifacts, and handoff notes.
  4. Admin controls Toggleable permissions, execution boundaries, audit visibility, and fail-safe stops.
